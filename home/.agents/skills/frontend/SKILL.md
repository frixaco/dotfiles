# Frontend

Use this skill for frontend implementation where interaction quality, visual quality, or component architecture materially affects the result.

## Pick The Mode

- Product-grade web UI, pages, apps, dashboards, posters, or components: read `references/design/guide.md`, then load specific design references only as needed.
- React component API design, composition, compound components, render props, context providers, or boolean-prop cleanup: read `references/composition/guide.md` and targeted files under `references/composition/rules/`.
- React Native or Expo UI/performance work: read `references/react-native/guide.md` and targeted files under `references/react-native/rules/`.
- React/Next.js View Transitions, shared element animations, route transitions, list reorder animation, or `startViewTransition`: read `references/view-transitions/guide.md`, then use targeted files in `references/view-transitions/`.

## Rules

- Build the usable experience first; do not default to a marketing landing page.
- Match the app's existing design system and implementation patterns.
- Use stable dimensions and responsive constraints so controls, boards, grids, and cards do not shift or overlap.
- Verify meaningful UI changes in a browser or simulator when a runnable target exists.
- Prefer proven libraries for complex domain logic, animation, and platform behavior unless the user asks for a from-scratch implementation.
